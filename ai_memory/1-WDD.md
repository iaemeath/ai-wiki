
## 📐 “多维视觉骨架与双轨业务减熵方案”（1-WDD）标准实施白皮书

### 暨 第一部分：确定系统（需求与设计对齐）全核核心框架

### 1. 方案背景与核心痛点

在项目生命周期的**第一阶段：确定系统**中，研发团队面临的最大敌人是“信息熵”。原始二级需求表描述极度模糊，导致以下工程硬伤：

- **文字说明书的“冷冰冰幻觉”：** 纯文字需求虽然字数多，但极度缺乏直观性。人类和 AI 都在脑补画面，往往系统开发过半，才发现前端交互或数据高亮完全做错。
    
- **内网重型原型的“高耦合死锁”：** 试图在内网构建包含路由、基座、后端网关的高保真原型，耗时极长，且多模块间极易产生代码语义的交叉污染。
    
- **“画页面”与“理业务”的本末倒置：** 团队花费大量精力去抠前端样式，却忽略了**底层的核心业务逻辑和算法边界（如省地通信通道、异常纠错熔断机制）才是系统的灵魂**。
    

### 2. 1-WDD 核心哲学

1-WDD 方案的核心哲学是：**“以隔离视觉骨架为催化剂，用最快的速度锁死业务说明书”**。

- **第一轨（多维隔离视觉骨架）：** 页面不是最终的生产代码，它的存在**纯粹是为了“看”和“辅助直观感受”**。必须生成极快、物理彻底隔离、井水不犯河水。
    
- **第二轨（关键业务说明书）：** 网页只是外壳，业务才是核心。利用看页面时被激发出的人类思维漏洞，反向压榨出高密度的关键业务说明书（ADR 架构决策、行为边界描述）。
    

### 3. 全栈工具链选择

- **业务减熵与代码直出层：Dify + 全量二级需求表 + GLM-5.2**
    
    一键导入整张表。利用大模型强大的长文本推理能力与代码高频直出能力，跳过任何架构分层，直接将各个功能模块吐成“面条式”的纯前端单文件组件代码。
    
- **秒级视觉催化层：内网离线沙箱（如 OpenWebUI Artifacts / vue-playground 离线版）**
    
    部署在内网的轻量级动态渲染运行时。它不与任何内网正式数据库、网关连接，只负责将大模型吐出的隔离单文件代码**在浏览器右侧秒级渲染成可以点击、高亮的视觉骨架**。
    

### 4. 标准流水线执行流程

#### 动作 1：物理隔离的面条式 Mock 页面秒级生成

GLM-5.2 针对二级需求表里的每一个行项目，在隔离应用模板的 `src/mock-views/` 目录下，生成**彼此完全独立、没有任何路由和状态依赖的单文件组件（Single File Component, .vue 或 .tsx）**。

- **机制：** 每个文件都是“面条代码”（HTML 结构、Vue 响应式变量、基础中台 UI 样式、以及模拟的本地 JSON 假数据**全部死死写在一个文件里**）。
    
- **优势：** 这种页面没有任何外部依赖耦合。AI 修改模块 A 时，绝对不可能改垮模块 B，生成和重构速度达到**秒级级响应**。
    

#### 动作 2：基于隔离骨架的业务缺陷反向排查

开发负责人打开内网离线沙箱浏览器，肉眼审查这些隔离的骨架页面。在真实的“看页面”过程中，由于视觉组件的直观刺激，人类的架构师思维会被瞬间激活，开始**反向捕捉原本被文字掩盖的业务漏洞**：

> _“我看到 AI 在‘省地流程同步’页面上画了一个撤销按钮。那么，如果省调那边由于网络原因导致撤销操作不成功，这个 Mock 页面上给出的‘失败提醒信息’是应该弹窗、还是写入历史日志？如果撤销成功了，本地台账的状态应该自动变更为什么值？”_

#### 动作 3：双轨事实源冻结与冷冻

人类将这些由视觉骨架逼出来的、真正决定系统灵魂的业务核心方案输入给大模型。大模型立即进行处理，产出“双轨事实源”并一键冻结：

1. **第一轨（多维隔离视觉骨架）：** 被锁定的、包含基础 UI 元素的静态 `.vue` 文件（作为后续开发的视觉图纸）。
    
2. **第二轨（关键业务说明书）：** 明确写死核心业务方案、异常流策略与 ADR 决策的 Markdown 文本（作为后续开发的逻辑图纸）。
    

### 5. 质量红线门禁（Quality Gates）

- **业务说明书“空值拦截门禁”：** 如果大模型生成的隔离 Mock 页面中包含任何带有“变更”、“提交”、“撤销”等写操作的按钮，那么《关键业务说明书》中**必须强行存在对应的 ADR 记录或异常流处置逻辑**。如果发现页面有按钮但说明书没有写逻辑方案，红线判定为不通过，拒绝向下游输送。
    
- **组件契约级覆盖检查：** 系统会自动 Lint 大模型吐出的隔离 Vue 文件，确保其调用的组件必须 100% 属于公司内网私服发布的标准中台组件库，防止大模型自行脑补不存在的私有 UI 标签。
    

### 💡 架构师总结

通过 **`1-WDD`** 方案，你在项目的第一阶段用最低的成本、最快的速度，把系统的“外壳（隔离骨架）”**和**“灵魂（关键业务说明书）”死死扣在手里。

当第一阶段输出被锁定后，你可以无缝将其作为最高输入，接入你定义的 **`2-DAO-AE`** 方案。由编排引擎将双轨资产切碎为 4 层拓扑依赖的 GitLab Issue 链，让 Claude Code 开启高确定性的原子级纯净编码。前后两手交织，构成了完美的工业级 AI 研发闭环。
