
# 🌐 全景深度分析：1-SMW + 2-DAO-AE 研发生态闭环

整套流水线的核心逻辑，是一个“由虚到实、信息密度逐级放大、确定性百分之百收敛”的精精密工厂。

```
【第一部分：1-SMW 确定系统】
  二级需求表 ──> (苏格拉底发问) ──> 锁定业务说明书 + 隔离骨架网页(.vue) + 逻辑图纸(Mermaid)
                                                                │
                                                           (作为最高输入)
                                                                ▼
【第二部分：2-DAO-AE 实现系统】
  注入【编码规范+脚手架】 ──> 技术拓扑拆解为 4 层 Issues ──> Claude Code (全库感知/离线编译) ──> 生产级代码
```

### 🧠 第一部分分析：1-SMW（用隔离视觉骨架作诱饵，榨出业务灵魂）

在第一阶段，大模型不写任何正式的后端，甚至不考虑数据库。

- 它的目标是**减熵**。通过“苏格拉底面板”强行让架构师在各种备选技术通道（如同步选 MQ 还是 REST）中做单选或填空，产生 **ADR（架构决策记录）**。
    
- 它利用 **1-WDD 隔离单文件网页** 让你快速肉眼审查界面，用视觉刺激你发现隐藏的业务漏洞（如：如果网络超时，页面怎么红字高亮？）。
    
- 它不画物理 ER 图，而是产出 **1-MDD 逻辑状态机与时序图纸（Mermaid）**。到此为止，系统的“灵魂（业务逻辑）”和“皮肤（视觉骨架）”被死死扣住。
    

### ⚙️ 第二部分分析：2-DAO-AE（注入编码规范，高确定性原子化砌砖）

当“一个准确描述的系统”输入到第二阶段后，核心使命变成了**不走样地翻译成代码**。

- **编码规范的注入：** 就在这个节点，流水线开始注入你们团队的 Java/Vue **编码规范、分层脚手架模板和数据库建表命名规范**。
    
- **拓扑切碎：** 编排引擎（LangGraph）将图纸和规范融合，切碎为 **[DDL构筑层] $\rightarrow$ [核心内核层] $\rightarrow$ [API边界层] $\rightarrow$ [正式前端重构层]** 这 4 层强依赖的 GitLab Issue 链。
    
- **极致爆破：** **Claude Code** 挂载内网基座 MCP，一次只读一个 Issue，在极小的上下文内依靠 GLM-5.2 的 Prompt Cache（提示词缓存）进行毫秒级高精度编码，并自动在本地运行 Maven/NPM 防御性编译测试，通过后一键 Push。
    

## 🛠️ 第一部分（1-SMW）：推荐开源组件库（内网拼装版）

第一部分的核心痛点是：**发问工作流控制、图纸即时渲染、隔离网页沙箱预览**。

### ❶ 核心发问与流控中心：Dify / FastGPT

- **定位：** 本地自托管的大模型工作流（Workflow）编排基座。
    
- **用法：** 利用 Dify 的 Chatflow（对话流）作为 1-SMW 的主 Web 界面。利用其 **Iterator（迭代器）** 节点逐行读取你的模糊需求 Excel 表，利用 **Question（用户交互）** 节点在发现逻辑断层时，自动在网页中央弹出表单或选择题卡住流程，等待你做出最高决策。
    

### ❷ 逻辑图纸实时渲染器：Mermaid.js / PlantUML Web JS

- **定位：** 纯文本 DSL 驱动的矢量图形渲染引擎。
    
- **用法：** 离线下载 `mermaid.min.js` 嵌入到你们系统的 Web 前端。GLM-5.2 在对齐完业务后，输出标准的 Mermaid 文本（如 `sequenceDiagram ...`），前端实时在网页右侧直接渲染出高清、可点击、可导出的时序图与状态机图。
    

### ❸ 物理隔离视觉沙箱：OpenWebUI (Artifacts 功能)

- **定位：** 目前全球开源界最强大的多模型 Web 交互及代码实时预览沙箱。
    
- **用法：** 完全离线 Docker 部署。利用其内置的 **Artifacts** 机制（类似 Anthropic Claude 的 Artifacts）。GLM-5.2 发问结束后，会直接直出 12 个模块互相隔离的“面条式”纯前端 `.vue` 或 `.html` 代码，OpenWebUI 会在网页侧边栏直接拉起隔离沙箱，**秒级渲染出可点击、带假数据的台账 Mock 网页**供人类肉眼审查。
    

## 🛠️ 第二部分（2-DAO-AE）：推荐开源组件库（确定性落地版）

第二部分的核心痛点是：**融入编码规范、零幻觉自动生成骨架、代码契约Lint硬拦截**。

### ❹ 确定性图纸/规范驱动脚手架：ZenWave 360

- **定位：** 面向领域驱动设计（DDD）的高精度、零幻觉代码生成引擎。
    
- **用法：** 在 2-DAO-AE 的技术拓扑拆解阶段，将 1-SMW 锁定的数据模型和 API 契约，结合你们公司的**编码规范模板**输入给 ZenWave。它能够 **100% 零幻觉、秒级一键生成** 所有的 Spring Boot 实体类（Entity）、Mapper 接口、持久层Repository 以及合规的数据库建表 DDL 骨架。大模型负责把框子搭好，绝不浪费算力。
    

### ❺ 架构规范与分层契约强拦截门禁：ArchUnit

- **定位：** 基于单元测试的 Java 架构规范与依赖关系 Lint 检查框架。
    
- **用法：** 这是用来**死锁大模型编码规范**的杀手锏。你可以让流水线在内网代码脚手架里内置几条 ArchUnit 规则，例如：
    
    Java
    
    ```
    classes().that().resideInAPackage("..controller..")
    .should().onlyCallMethodsThat().resideInAPackage("..service..");
    ```
    
    当 Claude Code 编写代码时，如果产生了任何不符合公司编码规范、不符合分层契约的“脑补行为”，在本地自动运行 `mvn test` 时，ArchUnit 会瞬间引爆编译报错并拦截 Git Push，强行把大模型的思维规范拉回正轨。
    

### ❻ 分布式任务状态机与研发生态纽带：GitLab Server + 社区版 Gitlab MCP Server

- **定位：** 物理 Issue 承载器与大模型连接内网研发生态的无缝协议桥梁。
    
- **用法：** 利用 LangGraph 调用 GitLab REST API，将切碎的 4 层任务灌入 GitLab 看板。本地开发人员启动 Claude Code 后，通过通过离线配置的 `mcp-server-gitlab`，允许 Claude Code 具备直接读取 Issue、查看上游依赖、在 Issue 评论区追加技术细节的双向联动能力。
